O que é um SGBD? Tipos e Aplicações?

SGBD significa Sistema de Gerenciamento de Banco de Dados (em inglês, Database Management System – DBMS).
Ele é um software responsável por criar, organizar, armazenar, manipular e administrar bancos de dados, Em outras palavras, o SGBD é o intermediário entre o usuário (ou o programa) e os dados, garantindo que as informações sejam acessadas com segurança, rapidez e consistência

Exemplos de SGBDs conhecidos incluem: MySQL, PostgreSQL, Oracle, SQL Server, MariaDB e SQLite.

Bancos de Dados Relacionais (SQL)

São os mais utilizados, organizam os dados em tabelas e utilizam a linguagem SQL para consultas, eles são utilizados em Sistemas empresariais, sites, e-commerce, sistemas que precisam de organização e relacionamentos entre dados.

Temos como exemplo:

  • MySQL

  • PostgreSQL

  • SQL Server

  • Oracle

  • MariaDB

Banco de Dados Não Relacionais (NoSQL)

Elas não usam o modelo tradicional de tabelas permitindo armazenar dados de maneira variada, São mais flexíveis e costumam oferecer melhor desempenho em grande escala, são usadas geralmente em aplicações que lidam com grandes volumes de dados, redes sociais, sistemas em tempo real, aplicações distribuídas.

Temos como exemplo:

  • MongoDB (JSON)

  • Redis

  • Cassandra

  • Neo4J

Banco de Dados na Nuvem

São serviços de banco de dados que rodam totalmente na nuvem, com manutenção automatizada, alta escalabilidade e tambem a redução de custos operacionais.

Temos como Exemplo:

  • Firebase Firestore

  • AWS RDS

  • Azure SQL

  • Supabase

Banco de Dados para Big Data

Criados para lidar com quantidades gigantescas de dados e análises complexas, usados geralmente em Análise de dados, inteligência artificial, processamento em larga escala, empresas com grandes volumes de informação.

Temos como Exemplo:

  • Hadoop HDFS

  • Big Query

  • SnowFlake

O que é um dicionario de Dados?

O dicionário de dados é um documento (ou módulo dentro do próprio banco de dados) que reúne todas as informações sobre os dados que existem em um sistema. Ele descreve os nomes das tabelas, seus campos, tipos de dados, tamanhos, relacionamentos, regras e restrições.
Em outras palavras, é como um manual que explica como o banco de dados está estruturado.
